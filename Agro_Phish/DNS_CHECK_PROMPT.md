# Промпт для ИИ: Описание работы кнопки "DNS проверка"

## Задача
Объясни, как работает кнопка "DNS проверка" в браузерном расширении PhishGuard. Нужно полное техническое описание со всеми деталями.

## Контекст
Это функция безопасности в Chrome расширении для защиты от фишинга. Кнопка анализирует DNS-записи домена текущей веб-страницы.

## Используемые инструменты и API

### 1. DNS-over-HTTPS (DoH) провайдеры:

**Cloudflare DNS** (основной):
- URL: `https://cloudflare-dns.com/dns-query?name={domain}&type={type}`
- Метод: GET
- Headers: `Accept: application/dns-json`
- Формат ответа: JSON (DNS-over-HTTPS JSON)
- Типы запросов: A (IPv4), AAAA (IPv6), MX (почтовые серверы)

**Google DNS** (резервный):
- URL: `https://dns.google/resolve?name={domain}&type={type}`
- Метод: GET
- Использование: Fallback при ошибке Cloudflare

### 2. IP Geolocation API:

**ipinfo.io**:
- URL: `https://ipinfo.io/{ip}/json`
- Возвращает: город, регион, страна, координаты, организация (ASN), hostname, флаг дата-центра

## Алгоритм работы

### Этап 1: Валидация
- Проверка, что URL HTTP/HTTPS
- Извлечение домена из `currentTab.url`
- Блокировка внутренних страниц браузера

### Этап 2: DNS запросы
Параллельно выполняются 3 запроса:
1. **A записи** (type=1) → IPv4 адреса
2. **AAAA записи** (type=28) → IPv6 адреса  
3. **MX записи** (type=15) → почтовые серверы

Механизм fallback: Cloudflare → Google DNS при ошибке

### Этап 3: Геолокация
- Берется первый IPv4 адрес
- Запрос к ipinfo.io для получения:
  - Геолокации (город, регион, страна)
  - Провайдера/организации (ASN)
  - Hostname сервера
  - Признаков дата-центра

### Этап 4: Анализ рисков
Функция `analyzeRisk(ipInfo, domain)` проверяет:

**Базы данных провайдеров:**
- `KNOWN_HOSTING_PROVIDERS`: amazon, aws, digitalocean, google, azure, cloudflare и др. (профессиональный хостинг = хорошо)
- `KNOWN_ISP_PROVIDERS`: comcast, verizon, rostelecom, beeline, megafon и др. (домашний провайдер = +30 к риску)

**Проверки:**
- VPN/Proxy индикаторы в org/hostname ("vpn", "proxy", "tunnel") → +20 к риску
- Совпадение hostname с доменом
- Страна сервера (подозрительные: RU, CN, KP, IR, BY)
- Флаг дата-центра (ipInfo.hosting)

**Расчет уровня риска:**
- 0-19 баллов → Низкий риск (🟢)
- 20-39 баллов → Средний риск (🟡)
- 40+ баллов → Высокий риск (🔴)

### Этап 5: Отображение результатов
UI показывает:
- Статус выполнения (индикатор + текст)
- Домен
- Список IP адресов (IPv4 и IPv6 с метками)
- MX записи (если есть)
- Геолокацию (город, регион, страна, координаты)
- Провайдера/организацию
- Бейдж уровня риска (цветной)
- Детали анализа (список найденных признаков)

## Структура данных

**DNS ответ:**
```json
{
  "Status": 0,
  "Answer": [{
    "name": "example.com",
    "type": 1,  // 1=A, 28=AAAA, 15=MX
    "TTL": 300,
    "data": "192.0.2.1"
  }]
}
```

**IP информация (ipinfo.io):**
```json
{
  "city": "Moscow",
  "region": "Moscow",
  "country": "RU",
  "loc": "55.7558,37.6173",
  "org": "AS12345 Example Hosting",
  "hostname": "server.example.com",
  "hosting": true
}
```

## Разрешения (manifest.json)
```json
{
  "host_permissions": [
    "https://cloudflare-dns.com/*",
    "https://dns.google/*",
    "https://ipinfo.io/*"
  ]
}
```

## Обработка ошибок
- DNS ошибка → fallback на другой провайдер
- IP info ошибка → продолжение без геолокации
- Общая ошибка → сообщение пользователю

## Технические детали
- Язык: JavaScript (ES6+)
- API: Fetch API
- Асинхронность: async/await
- Формат: JSON
- UI: Прямая манипуляция DOM

## Цель функции
Выявление признаков фишинга через анализ DNS инфраструктуры:
- Проверка подлинности домена
- Обнаружение подозрительных хостинг-провайдеров
- Выявление VPN/Proxy серверов
- Определение географического расположения
- Оценка общего уровня риска
