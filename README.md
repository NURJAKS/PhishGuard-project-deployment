### 📝 Краткое описание

**PhishGuard AI** — это система защиты от мошеннических сайтов, которая работает как умный охранник в вашем браузере. Она автоматически проверяет каждый сайт, который вы открываете, и предупреждает вас, если сайт подозрительный или опасный.

**Что умеет система:**
- ✅ Автоматически блокирует опасные сайты
- ✅ Проверяет страницы оплаты, чтобы защитить ваши данные карты
- ✅ Использует искусственный интеллект для определения поддельных сайтов
- ✅ Проверяет документы на наличие мошеннических схем
- ✅ Показывает понятные объяснения, почему сайт опасен
- ✅ Работает на русском, казахском и английском языках

**Для кого:**
- Обычные пользователи интернета
- Банки и финансовые компании
- Бизнес и организации
- Все, кто хочет защититься от мошенников в интернете

## 🌟 Обзор

**PhishGuard AI** — это умная система защиты от мошеннических сайтов в интернете. Представьте, что у вас есть личный охранник, который проверяет каждый сайт, который вы посещаете, и предупреждает вас, если что-то подозрительно.

### 🎯 Что это такое простыми словами?

PhishGuard — это программа, которая:
- **Проверяет сайты** на подозрительность, пока вы их посещаете
- **Блокирует опасные сайты** автоматически, чтобы вы не попались на удочку мошенников
- **Использует искусственный интеллект** для понимания, является ли сайт подделкой
- **Анализирует страницы оплаты**, чтобы убедиться, что ваши данные карты не попадут к мошенникам
- **Проверяет документы** на наличие подозрительных ссылок или мошеннических схем

### 🎯 Для кого это нужно?

Система защищает:
- **Обычных пользователей** — от поддельных сайтов банков, соцсетей и других сервисов
- **Банки и финансовые компании** — от мошенников, которые создают копии их сайтов
- **Бизнес** — от утечки данных сотрудников через фишинговые сайты
- **Организации** — от кибератак через поддельные сайты

### 🏛️ Как это работает?

Система состоит из нескольких частей, которые работают вместе:

- **Расширение для браузера** — устанавливается в Chrome и работает в фоне, проверяя все сайты, которые вы открываете
- **Сервер проверки** — это "мозг" системы, который анализирует сайты и принимает решения
- **Искусственный интеллект** — использует умные алгоритмы Google и других сервисов, чтобы понять, является ли сайт подделкой
- **Инструменты безопасности** — специальные программы для глубокой проверки сайтов на уязвимости
- **Панель статистики** — показывает, сколько опасных сайтов было заблокировано и другую полезную информацию
- **База данных** — хранит информацию о всех проверенных сайтах

### 🔄 Версии проекта

Проект включает несколько реализаций для разных сценариев использования:

1. **Agro_Phish** (Основная версия) - Python/FastAPI с полным функционалом
2. **PhishGuardCSharp** - ASP.NET Core версия для .NET экосистемы
3. **PhishGuardAI** - AWS Lambda версия для serverless архитектуры
4. **Pinkerton Integration** - Специализированный движок для сканирования секретов в JavaScript

### 🎯 Основные возможности

#### 🤖 Умный анализ сайтов
- **Искусственный интеллект** — система использует самые современные AI-технологии для понимания, является ли сайт подделкой
- **Сравнение с настоящими сайтами** — система знает, как выглядят настоящие сайты банков и других компаний, и сравнивает с ними подозрительные сайты
- **Понятные объяснения** — система объясняет простыми словами, почему сайт опасен, без сложных технических терминов
- **Полная проверка** — анализируется всё: адрес сайта, защита соединения, содержимое страницы, куда отправляются ваши данные
- **Оценка риска** — система показывает, насколько опасен сайт (высокий, средний или низкий риск)

#### 🛡️ Автоматическая защита
- **Мгновенная блокировка** — опасные сайты блокируются сразу, как только система их обнаруживает
- **Списки безопасных и опасных сайтов** — вы можете добавить сайты, которым доверяете, или заблокировать подозрительные
- **Автоматическая проверка** — система может проверять все сайты, которые вы посещаете, без вашего участия
- **Уведомления** — система отправляет подробные отчеты о заблокированных сайтах в Telegram
- **Работает незаметно** — расширение для браузера работает в фоне, не мешая вам

#### 🔍 Глубокая проверка безопасности
- **Проверка кода сайта** — система ищет в коде сайта утечки паролей, ключей доступа и другой секретной информации
- **Сканирование портов** — проверяет, какие сервисы открыты на сервере сайта
- **Поиск скрытых файлов** — находит скрытые страницы и файлы, которые мошенники не хотят показывать
- **Проверка уязвимостей** — ищет известные проблемы безопасности на сайте

#### 💳 Защита при оплате
- **Проверка страниц оплаты** — система специально проверяет страницы, где вы вводите данные карты
- **Скрытие номеров карт** — номера карт автоматически скрываются для безопасности
- **Проверка платежных систем** — система знает настоящие адреса платежных систем (Stripe, PayPal и др.) и проверяет, что вы на правильном сайте
- **Проверка, куда идут данные** — система проверяет, что ваши данные отправляются на настоящий сайт, а не мошенникам
- **Поддержка разных языков** — распознает формы оплаты на русском, казахском и английском языках

#### 📄 Проверка документов
- **Разные форматы** — проверяет PDF, Word, Excel и другие документы
- **Поиск опасной информации** — находит номера карт, банковские счета и другую чувствительную информацию
- **Проверка на мошенничество** — анализирует документы на признаки подделки или мошеннических схем
- **Безопасный обмен** — создает безопасные ссылки для обмена документами

#### 🧾 Проверка счетов
- **Проверка подлинности** — система проверяет, настоящий ли счет или поддельный
- **Проверка реквизитов** — проверяет банковские реквизиты, ИНН, БИН на правильность
- **Поиск дубликатов** — находит, если один и тот же счет проверялся несколько раз
- **Рекомендации** — дает советы, что делать, если счет подозрительный

#### 📊 Статистика и отчеты
- **Красивые графики** — показывает статистику в виде понятных графиков и диаграмм
- **История проверок** — сохраняет все проверенные сайты, чтобы вы могли посмотреть историю
- **Фильтры и поиск** — можно искать и фильтровать проверенные сайты по разным критериям
- **Экспорт данных** — можно выгрузить статистику для отчетов

#### 🌐 Поддержка разных языков
- **Русский, казахский, английский** — система работает на этих языках
- **Понятные объяснения** — объяснения написаны простым языком, без сложных терминов
- **Распознавание на разных языках** — находит подозрительные элементы на разных языках

### 🔧 Технические компоненты

#### Backend (FastAPI)
- **Язык**: Python 3.10+
- **Фреймворк**: FastAPI 0.104+ с автоматической документацией Swagger
- **База данных**: SQLite с SQLAlchemy ORM
- **Модели данных**: 
  - `Incident` - записи о проверенных URL с результатами анализа
  - `PaymentCheck` - результаты анализа платежных страниц
  - `InvoiceCheck` - результаты проверки счетов-фактур
- **API версионирование**: RESTful API с версионированием через `/v1/`
- **CORS**: Настроен для работы с Chrome Extension
- **Логирование**: Структурированное логирование всех операций

#### Chrome Extension (Manifest V3)
- **Тип**: Service Worker (background.js) для фоновой работы
- **Content Scripts**: Инъекция скриптов на все страницы для мониторинга
- **Popup Interface**: Пользовательский интерфейс для ручных проверок
- **Dashboard**: Встроенная страница статистики
- **Admin Panel**: Панель управления черными/белыми списками
- **Document Viewer**: Просмотр и анализ загруженных документов
- **Invoice Viewer**: Специализированный просмотрщик счетов

#### AI Engine
- **Основной провайдер**: Google Generative AI (Gemini 2.5 Flash)
- **Резервный провайдер**: OpenRouter с поддержкой различных моделей
- **Промпты**: Специально разработанные промпты для простого языка объяснений
- **Сравнение источников**: Интеграция с базой `official_sources.json` для сравнения с оригинальными сайтами
- **Fallback механизм**: Автоматическое переключение между провайдерами при ошибках

#### Security Tools
- **Masscan**: Быстрое сканирование портов (1-65535) с использованием raw sockets
- **Gobuster**: Поиск скрытых директорий и файлов на веб-серверах
- **Nikto**: Комплексное сканирование веб-серверов на известные уязвимости
- **Pinkerton**: Специализированный движок для обнаружения секретов в коде

#### Database Schema
```sql
-- Инциденты (проверенные URL)
incidents: id, url, action, score, reason, timestamp

-- Проверки платежных страниц
payment_checks: id, request_id, url, domain, safe, score, reasons, html_hash, timestamp

-- Проверки счетов-фактур
invoice_checks: id, analysis_id, filename, doc_hash, status, score, invoice_data, checks, reasons, recommendations, timestamp
```

#### Конфигурационные файлы
- `rules.json` - черные/белые списки, подозрительные ключевые слова, TLD
- `payment_rules.json` - правила анализа платежных страниц с весами и порогами
- `official_sources.json` - база официальных доменов банков, платежных систем, IT-компаний
- `auto_scan_config.json` - настройки автоматического сканирования
- `.env` - API ключи для AI-провайдеров (опционально)

## 🚀 Установка

> 💡 **Быстрый старт**: Для быстрой установки см. [INSTALLATION.md](INSTALLATION.md)

### 1. Клонирование репозитория

```bash
git clone https://github.com/Nurbol876047/Phishguard_GOBUSTER.git
cd Phishguard_GOBUSTER
```

### 2. Установка зависимостей Backend

```bash
cd Agro_Phish/backend

# Создание виртуального окружения
python3 -m venv venv

# Активация виртуального окружения
# Linux/macOS:
source venv/bin/activate
# Windows:
# venv\Scripts\activate

# Установка зависимостей
pip install -r requirements.txt
```

### 3. Настройка API ключей (опционально)

Для использования AI-анализа создайте файл `.env` в `Agro_Phish/backend/`:

```bash
cd Agro_Phish/backend
cat > .env << EOF
GOOGLE_AI_API_KEY=your_google_ai_api_key_here
OPENROUTER_API_KEY=your_openrouter_api_key_here
EOF
```

**Примечание**: AI-анализ работает и без API ключей, но с ограниченной функциональностью.

### 4. Настройка прав для Masscan

Masscan требует специальных прав для работы с raw sockets:

```bash
cd Agro_Phish
chmod +x setup_masscan_permissions.sh
sudo ./setup_masscan_permissions.sh
```

Или вручную:

```bash
sudo setcap cap_net_raw,cap_net_admin=eip Agro_Phish/tools/masscan/masscan
```

Проверка прав:

```bash
getcap Agro_Phish/tools/masscan/masscan
```

Должно вывести: `masscan = cap_net_admin,cap_net_raw+eip`

### 5. Установка зависимостей Dashboard (опционально)

```bash
cd Agro_Phish/dashboard
pip install -r requirements.txt
```

## ▶️ Запуск

### 1. Запуск Backend сервера

```bash
cd Agro_Phish/backend
source venv/bin/activate  # Linux/macOS
# или: venv\Scripts\activate  # Windows

python3 -m uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

Backend будет доступен на: **http://localhost:8000**

**Проверка работоспособности:**

```bash
curl http://localhost:8000/health
```

Должен вернуть: `{"status":"healthy","timestamp":"..."}`

### 2. Установка Chrome Extension

1. Откройте Chrome и перейдите на `chrome://extensions/`
2. Включите **"Режим разработчика"** (Developer Mode)
3. Нажмите **"Загрузить распакованное расширение"** (Load unpacked)
4. Выберите папку: `Agro_Phish/extension/`
5. Расширение установлено! Иконка щита появится в панели инструментов

### 3. Запуск Dashboard (опционально)

```bash
cd Agro_Phish/dashboard
streamlit run app.py --server.port 8501
```

Dashboard будет доступен на: **http://localhost:8501**

#### Проверка URL

```bash
curl -X POST http://localhost:8000/v1/check/url \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com"}'
```

```bash
curl -X POST http://localhost:8000/v1/ai/scan \
```


```bash
curl -X POST http://localhost:8000/v1/vuln/gobuster \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com"}'
```

## 💡 Примеры использования и сценарии

### Сценарий 1: Защита пользователя от фишингового сайта

1. Пользователь посещает подозрительный сайт
2. Chrome Extension автоматически отправляет URL на проверку
3. Backend анализирует URL по правилам (черный список, паттерны)
4. Если сайт подозрителен, выполняется AI-анализ
5. AI сравнивает сайт с официальными источниками
6. Если обнаружен фишинг:
   - Сайт блокируется
   - Пользователь видит предупреждение
   - Домен автоматически добавляется в черный список
   - Отправляется детальный отчет в Telegram

### Сценарий 2: Проверка платежной страницы

1. Пользователь открывает страницу оплаты
2. Extension перехватывает HTML формы
3. Backend анализирует:
   - Наличие полей карты
   - URL действия формы (куда отправляются данные)
   - HTTPS протокол
   - Соответствие домену платежной системы
4. Выполняется AI-анализ страницы
5. Результат отображается пользователю с объяснением простым языком

### Сценарий 3: Анализ подозрительного документа

1. Пользователь загружает PDF документ через Extension
2. Backend извлекает текст из документа
3. Анализируется на наличие:
   - Номеров карт (автоматически маскируются)
   - Банковских реквизитов
   - Подозрительных ссылок
   - Признаков фишинга
4. Результат сохраняется с уникальным ID
5. Пользователь получает публичную ссылку для обмена

### Сценарий 4: Проверка счета-фактуры

1. Бухгалтер загружает счет через веб-интерфейс
2. AI анализирует документ на:
   - Подлинность реквизитов
   - Соответствие формату
   - Валидность БИН/ИНН
   - Признаки подделки
3. Система выдает вердикт: accepted/suspicious/rejected
4. Сохраняется история проверок для аудита
5. Генерируются рекомендации по безопасности

### Сценарий 5: Аудит безопасности сайта

1. Администратор запускает полный аудит через API
2. Система собирает данные:
   - DNS информация (IP, страна, организация)
   - TLS сертификат (валидность, издатель, срок)
   - Возраст домена
   - HTML структура
   - JavaScript файлы
3. AI выполняет комплексный анализ всех аспектов
4. Генерируется детальный отчет с:
   - Анализом каждого компонента
   - Вероятностью фишинга (0-100%)
   - Итоговым вердиктом
   - Простыми объяснениями

### Сценарий 6: Мониторинг и аналитика

1. Администратор открывает Streamlit Dashboard
2. Просматривает статистику:
   - Общее количество проверок
   - Процент заблокированных сайтов
   - Тренды по времени
   - Топ подозрительных доменов
3. Фильтрует инциденты по типу, дате, домену
4. Экспортирует данные для отчетности

## 🐛 Устранение неполадок

### Backend не запускается

**Проблема**: Порт 8000 занят

**Решение**:
```bash
# Найти процесс на порту 8000
lsof -i :8000  # Linux/macOS
netstat -ano | findstr :8000  # Windows

# Остановить процесс
kill -9 <PID>  # Linux/macOS
taskkill /PID <PID> /F  # Windows
```

### Masscan не работает

**Проблема**: `Masscan requires root privileges`

**Решение**:
```bash
cd Agro_Phish
sudo ./setup_masscan_permissions.sh
```

Или вручную:
```bash
sudo setcap cap_net_raw,cap_net_admin=eip Agro_Phish/tools/masscan/masscan
```

### Chrome Extension не подключается к Backend

**Проблема**: Расширение не может связаться с API

**Решение**:
1. Убедитесь, что backend запущен на порту 8000
2. Проверьте `chrome://extensions/` → PhishGuard → "Проверить доступ к URL-адресам"
3. Проверьте консоль расширения: `chrome://extensions/` → PhishGuard → "Просмотр сервис-воркера"

### Ошибки AI-анализа

**Проблема**: AI-анализ не работает

**Решение**:
- AI-анализ опционален и работает без API ключей с ограничениями
- Для полной функциональности настройте `.env` файл с API ключами
- Проверьте логи backend для деталей ошибок

### База данных не создается

**Проблема**: Ошибки при работе с БД

**Решение**:
```bash
cd Agro_Phish/backend
rm -rf database/phishguard.db  # Удалить старую БД
# Перезапустить backend - БД создастся автоматически
```

## 🔒 Безопасность и лучшие практики

### Рекомендации по развертыванию

#### Production окружение
1. **API ключи**: Никогда не коммитьте API ключи в репозиторий. Используйте переменные окружения или секретные менеджеры
2. **HTTPS**: Всегда используйте HTTPS для production backend
3. **CORS**: Ограничьте CORS только необходимыми доменами
4. **Rate Limiting**: Добавьте rate limiting для защиты от злоупотреблений
5. **Логирование**: Настройте централизованное логирование (ELK, Splunk)
6. **Мониторинг**: Используйте мониторинг (Prometheus, Grafana) для отслеживания метрик

#### База данных
1. **Резервное копирование**: Настройте регулярные бэкапы SQLite базы
2. **Миграции**: Используйте миграции для изменений схемы БД
3. **Индексы**: Убедитесь, что индексы настроены для частых запросов

#### Chrome Extension
1. **Манифест**: Проверьте все разрешения в manifest.json
2. **Content Security Policy**: Соблюдайте CSP для безопасности
3. **Обновления**: Регулярно обновляйте расширение с исправлениями безопасности

### Обработка чувствительных данных

#### Маскирование данных
- Номера карт (PAN) автоматически маскируются перед сохранением
- HTML-содержимое не сохраняется в БД, только хеш
- Секреты в JavaScript не сохраняются, только факт обнаружения

#### Хранение данных
- Документы хранятся с уникальными ID, без связи с пользователями
- Метаданные документов хранятся отдельно от содержимого
- История проверок может быть очищена через API

### API безопасность

#### Аутентификация (для production)
Рекомендуется добавить:
- API ключи для доступа к API
- OAuth 2.0 для пользовательской аутентификации
- JWT токены для сессий

#### Валидация входных данных
- Все URL валидируются перед обработкой
- Размер файлов ограничен (10MB для документов)
- Таймауты для всех внешних запросов
- Санитизация всех пользовательских входных данных

### Мониторинг и алертинг

#### Рекомендуемые метрики
- Количество проверок в минуту/час
- Процент заблокированных сайтов
- Время ответа API
- Ошибки AI-провайдеров
- Использование ресурсов (CPU, память, диск)

#### Алерты
- Критическое количество заблокированных сайтов
- Ошибки подключения к AI-провайдерам
- Превышение лимитов API
- Аномальная активность

### Обновление и поддержка

#### Регулярные обновления
1. **Правила**: Регулярно обновляйте `rules.json` с новыми паттернами
2. **Официальные источники**: Обновляйте `official_sources.json` с новыми банками/сервисами

### Производительность

#### Ограничения и лимиты
- **Размер HTML**: Максимум 500KB для анализа (обрезается автоматически)
- **Размер JS файлов**: Максимум 750KB на файл
- **Размер документов**: Максимум 10MB
- **Количество URL в AI-сканировании**: До 5 URL за запрос
- **Таймауты**: 
  - HTTP запросы: 8-12 секунд
  - AI анализ: 30 секунд
  - Port scan: 120 секунд
  - Gobuster: 120 секунд

#### Оптимизации
- Кэширование результатов проверок
- Параллельная обработка множественных запросов
- Асинхронная обработка через FastAPI async/await
- Ленивая загрузка больших файлов
